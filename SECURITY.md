# Security Policy

## Core principles

- Do not log API keys, tokens, passwords, secrets, or private keys.
- Telemetry defaults to metadata only: token counts, costs, model names, latency, routing decisions.
- Prompt and response bodies are not stored in cost telemetry by default.
- High-risk tool actions must be explicit and reviewable.

## Reporting

During private development, report security issues directly to the project owner. Public vulnerability reporting process will be defined before public alpha.
